# 前端架构 - 技术选型

## 技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| 原生 HTML5 | - | 页面结构 |
| 原生 CSS3 | - | 样式与主题（CSS 变量方案） |
| 原生 JavaScript (ES6+) | - | 交互逻辑 |
| Marked.js | v11+ | Markdown → HTML 渲染 |
| Highlight.js | v11+ | 代码语法高亮 |

## 为什么选择原生技术？

### 优势

1. **零构建工具**：无需 webpack、vite 等构建工具
2. **零框架依赖**：不依赖 React、Vue 等框架，减少体积
3. **直接运行**：浏览器直接解析，无需编译步骤
4. **易于理解**：代码直观，便于维护和修改

### 文件结构

```
index.html
├── <head>
│   ├── CDN 引入 Marked.js
│   ├── CDN 引入 Highlight.js
│   └── <style> CSS 样式（含 CSS 变量主题）
└── <body>
    ├── 侧边栏（目录树 + 工具栏）
    ├── 主内容区（Markdown 渲染/编辑器）
    └── 模态框（创建/删除/重命名/链接插入）
```

## 前端模块划分

| 模块 | 核心函数 | 功能 |
|------|---------|------|
| 目录树 | `renderTree()` | 递归渲染多级目录 |
| 文件浏览 | `loadFile()` | Markdown 渲染与显示 |
| 编辑 | `editFile()` / `saveFile()` | 源码编辑与保存 |
| 主题 | `toggleTheme()` | 白天/黑夜切换 |
| 链接插入 | `showLinkModal()` | 图形化插入内部链接 |
