### 问题 1：初始版本无层级结构

**分类**：架构设计

**现象**：
- 初始 Wiki 是平级文档列表，没有多层级目录结构
- 用户期望：`个人知识库 → AI Native Builder 知识库 → Clowder AI 架构 Wiki`

**解决方案**：

1. **目录结构重组**：将文档按层级放到对应文件夹中
   ```
   D:\文档\wiki\个人知识库\
     └── ai-native-builder知识库\
           └── clowder-ai架构wiki\
                 ├── 总目录.md
                 └── ...
   ```

2. **前端改造**：目录树从平级列表改为递归树状导航

3. **后端改造**：`scanDirectory()` 函数从返回平级列表改为递归构建树结构（`children` 字段）

### 问题 2：Windows 路径反斜杠导致 JSON 解析失败

**分类**：跨平台兼容

**现象**：
- `GET /api/wiki/tree` 返回的 JSON 中包含反斜杠 `\`
- 浏览器 JSON 解析报错：`Invalid \escape`
- `loadFile()` 请求的文件路径中包含反斜杠，服务器找不到文件

**根本原因**：
- Windows 的 `path.relative()`、`path.join()` 等函数返回使用反斜杠 `\` 作为分隔符
- JSON 中 `\` 是转义字符，`\文` 等序列导致非法转义

**解决方案**：

```javascript
// 所有返回给前端的路径统一替换为正斜杠
const relativePath = path.relative(baseDir, fullPath).replace(/\\/g, '/');
const fullPath_safe = fullPath.replace(/\\/g, '/');
```

**影响范围**：
- `scanDirectory()` 函数
- `resolveMarkdownLinks()` 函数
- 所有 API 响应中的路径字段

### 问题 3：内部链接跳转 404

**分类**：链接处理

**现象**：
- 文档中的 `[文本](./路径.md)` 链接点击后报 404
- 多次修复仍反复出现

**根因分析**：

#### 3.1 URL 编码问题

服务器 `resolveMarkdownLinks` 对路径做了 `encodeURIComponent()`，导致中文路径被编码为 `%E4%B8%AA...`。前端 `marked.js` 渲染后又做了一次编码，路径变成 `%25E4%B8%25AA...`。

**解决**：服务器不编码中文路径，直接返回原始中文。

```javascript
// 错误
return `[${text}](?file=${encodeURIComponent(relativePath)})`;

// 正确
return `[${text}](?file=${relativePath})`;
```

#### 3.2 Marked.js 渲染器 API 变更

`marked` 新版本中 `renderer.link(href, title, text)` 改为 `renderer.link(token)`。

```javascript
// 旧版本
renderer.link = function(href, title, text) { ... }

// 新版本
renderer.link = function(token) {
    const href = token.href;
    const title = token.title;
    const text = token.text;
    // ...
}
```

#### 3.3 `loadFile` 重复编码

`onclick` 中路径被 `encodeURIComponent` + `decodeURIComponent` 处理，引号转义不完整。

**解决**：直接传原始路径，只对 HTML 属性中的引号做转义。

```javascript
const safePath = linkPath.replace(/\\/g, '\\\\').replace(/'/g, "\\'");
return `<a href="javascript:void(0)" onclick="loadFile('${safePath}')">...</a>`;
```

#### 3.4 `loadFile` 缓存问题

`loadFile` 开头有 `if (currentFile === filePath) return;`，导致编辑模式返回浏览时不刷新。

**解决**：添加 `previewFile()` 函数，在调用 `loadFile` 前重置 `currentFile = null`。

### 问题 4：新建文件功能不工作

**分类**：后端 API

**现象**：
- 点击"新建文件"后没有实际创建文件
- 需要手动选择父目录，交互不友好

**解决方案**：

1. **服务器添加 `.md` 扩展名**：`POST /api/wiki/create` 自动为 `type: 'file'` 的创建添加 `.md`
2. **服务器自动创建父目录**：`if (!fs.existsSync(parentPath)) fs.mkdirSync(parentPath, { recursive: true })`
3. **前端自动使用当前目录**：用 `currentDir` 变量跟踪当前所在目录，创建时传入 `parentDir`

### 问题 5：编辑模式返回浏览无效

**分类**：前端逻辑

**现象**：
- 编辑器中点击"返回浏览"按钮，仍留在编辑页面
- "取消"按钮也有同样问题

**根本原因**：

`loadFile(filePath)` 开头有缓存检查：
```javascript
async function loadFile(filePath) {
    if (currentFile === filePath) return;  // <-- 直接返回，页面不刷新
}
```

编辑模式下 `currentFile` 没有被重置，所以 `loadFile` 什么都不做。

**解决方案**：

添加 `previewFile()` 函数，重置 `currentFile` 后重新加载：
```javascript
function previewFile(filePath) {
    currentFile = null;
    loadFile(filePath);
}
```

### 问题 6：README 链接指向错误

**分类**：数据修正

**现象**：
- `个人知识库/README.md` 中"Clowder AI 架构 Wiki"链接指向 `clowder-ai架构wiki/.md`
- 正确的目标应该是 `clowder-ai架构wiki/总目录.md`

**根本原因**：
- Markdown 链接写为 `[Clowder AI 架构 Wiki](./clowder-ai架构wiki/)`
- `resolveMarkdownLinks` 将目录路径补上 `.md` 后缀，成了 `clowder-ai架构wiki/.md`

**解决方案**：

将链接改为明确指向文件：
```markdown
[Clowder AI 架构 Wiki](?file=个人知识库/ai-native-builder知识库/clowder-ai架构wiki/总目录.md)
```

### 问题 7：Highlight.js 模块冲突

**分类**：CDN 兼容

**现象**：
```
Uncaught ReferenceError: module is not defined
Uncaught SyntaxError: Identifier 'IDENT_RE' has already been declared
```

**根本原因**：
- jsdelivr CDN 的 `lib/core.min.js` 和 `lib/languages/*.min.js` 是 CommonJS 模块
- 浏览器不识别 `module.exports`
- 多个语言模块并行加载时重复声明相同变量

**尝试过的方法**（均失败）：
1. 动态 `fetch` + `new Function` 包装 —— `hljs is not defined`
2. 尝试 `lib/highlight.min.js` —— 文件不存在（404）
3. 尝试 `build/` 目录 —— 文件不存在（404）

**最终解决方案**：

改用 **cdnjs** 的浏览器兼容版本：
```html
<script src="https://cdnjs.cloudflare.com/ajax/libs/highlight.js/11.9.0/highlight.min.js"></script>
```

cdnjs 的 bundle 内置常用语言，无需单独加载语言模块。

### 问题 8：编辑器代码块按钮报错

**分类**：HTML 模板转义

**现象**：
```
Uncaught SyntaxError: Invalid or unexpected token
```

**根本原因**：

模板字符串中生成的 HTML 包含反引号，与 JavaScript 模板字符串冲突：
```javascript
contentArea.innerHTML = `
    <button onclick="insertText('${'```\n代码块\n```'}')">
    //                                       ↑ 反引号破坏了 HTML 属性引号
`;
```

**解决方案**：

改为调用函数，在函数内部拼接字符串：
```javascript
<button onclick="insertCodeBlock()"><code>{ }</code></button>

function insertCodeBlock() {
    const code = '```\n代码块\n```';
    insertText(code);
}
```

### 问题 9：`../` 相对路径链接不转换

**分类**：正则表达式

**现象**：
- 文档中的 `[返回](../总目录.md)` 链接渲染后仍为 Markdown 源码
- `./` 开头的链接正常转换

**根本原因**：

`resolveMarkdownLinks` 正则只匹配 `./` 开头：
```javascript
return content.replace(/\[([^\]]+)\]\(\.\/([^)]+?)\)/g, ...);
```

**解决方案**：

扩展正则同时匹配 `./` 和 `../`：
```javascript
return content.replace(/\[([^\]]+)\]\((\.\/[^)]+|\.\.\/[^)]+)\)/g, ...);
```

### 问题 10：创建位置需手动选择目录

**分类**：UX 设计

**现象**：
- 新建文件/目录时需要从下拉菜单选择父目录
- 操作繁琐，不符合直觉

**解决方案**：

1. 添加 `currentDir` 变量追踪当前位置
2. 点击目录时更新 `currentDir = path`
3. 点击文件时更新 `currentDir = 文件所在目录`
4. 新建模态框显示"将在 xxx 目录下创建"，自动传入 `parentDir: currentDir`
5. 移除目录选择下拉框

## 技术经验总结

### 路径处理

- **Windows 路径统一转正斜杠**：`.replace(/\\/g, '/')`
- **内部链接不编码中文**：避免二次编码问题
- **路径安全检查**：`path.resolve()` + `startsWith()` 防止目录遍历

### HTML 模板字符串

- `innerHTML` 中包含反引号时需要转义或改用函数调用
- 路径中包含单引号时需要转义：`.replace(/'/g, "\\'")`

### CDN 选型

- CommonJS 模块不能直接在浏览器中使用
- 优先选择 CDN 的浏览器 bundle 版本
- cdnjs 通常比 jsdelivr 提供更多的浏览器兼容版本

### 状态管理

- 页面刷新时需要考虑状态重置
- 编辑模式与浏览模式切换需要清理缓存变量
- localStorage 用于持久化主题等用户偏好

### 正则表达式

- 相对路径的匹配需要覆盖 `./`、`../` 及更深层级的 `../../`
- 使用 `[^)]+` 替代 `.+?` 避免跨括号匹配
