# 后端架构 - 安全路径检查

## 核心函数

```javascript
function getSafePath(filePath) {
    // 拼接完整路径
    const fullPath = path.join(WIKI_DIR, filePath);

    // 解析绝对路径（消除 .. 等相对路径）
    const resolvedPath = path.resolve(fullPath);
    const resolvedWikiDir = path.resolve(WIKI_DIR);

    // 安全检查：确保在 Wiki 目录内
    if (!resolvedPath.startsWith(resolvedWikiDir)) {
        return null;  // 路径越界，拒绝访问
    }

    return fullPath;
}
```

## 安全威胁防护

### 1. 目录遍历攻击

**攻击示例**：
```
请求: GET /api/wiki/read?file=../../../../etc/passwd
拼接: D:\文档\wiki\../../../../etc/passwd
解析: C:\etc\passwd
检查: C:\etc\passwd 不以 D:\文档\wiki 开头
结果: 拒绝访问（返回 403）
```

**防护原理**：
```javascript
const resolvedPath = path.resolve(fullPath);  // C:\etc\passwd
const resolvedWikiDir = path.resolve(WIKI_DIR);  // D:\文档\wiki

// C:\etc\passwd.startsWith("D:\文档\wiki") === false
// → 返回 null → 返回 403 Forbidden
```

### 2. 空路径检查

```javascript
if (!filePath) {
    res.writeHead(400);
    res.end(JSON.stringify({ error: 'File parameter is required' }));
    return;
}
```

### 3. 文件存在性检查

```javascript
if (!fs.existsSync(fullPath)) {
    res.writeHead(404);
    res.end(JSON.stringify({ error: 'File not found' }));
    return;
}
```

## 所有 API 的安全检查

| API | 安全检查 |
|-----|---------|
| `GET /api/wiki/read` | `getSafePath()` + 文件存在检查 |
| `POST /api/wiki/create` | `getSafePath()` + 已存在检查 |
| `PUT /api/wiki/update` | `getSafePath()` + 文件存在检查 |
| `DELETE /api/wiki/delete` | `getSafePath()` + 文件存在检查 |
| `PUT /api/wiki/rename` | `getSafePath()`（新旧路径都检查） |

## CORS 配置

```javascript
res.setHeader('Access-Control-Allow-Origin', '*');
res.setHeader('Access-Control-Allow-Methods', 'GET, POST, PUT, DELETE, OPTIONS');
res.setHeader('Access-Control-Allow-Headers', 'Content-Type');
```

注意：当前配置允许所有来源，生产环境应限制为特定域名。
