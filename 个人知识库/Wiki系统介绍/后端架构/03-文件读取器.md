# 后端架构 - 文件读取器

## 核心函数

### 读取文件

```javascript
if (pathname === '/api/wiki/read' && method === 'GET') {
    const filePath = parsedUrl.searchParams.get('file');

    // URL 解码路径
    const decodedPath = decodeURIComponent(filePath);
    const fullPath = getSafePath(decodedPath);

    // 读取文件
    fs.readFile(fullPath, 'utf8', (err, data) => {
        // 解析 Markdown 内部链接
        const processedContent = resolveMarkdownLinks(data, decodedPath);
        res.end(processedContent);
    });
}
```

### 解析内部链接

```javascript
function resolveMarkdownLinks(content, currentFilePath) {
    const currentDir = path.dirname(currentFilePath);

    return content.replace(/\[([^\]]+)\]\(\.\/([^)]+?)\)/g, (match, text, linkPath) => {
        // 添加 .md 扩展名
        let resolvedPath = linkPath;
        if (!linkPath.endsWith('.md')) {
            resolvedPath += '.md';
        }

        // 解析相对路径为绝对路径
        const fullPath = path.resolve(currentDir, resolvedPath);
        const relativePath = path.relative(WIKI_DIR, fullPath).replace(/\\/g, '/');

        // 转换为前端路由格式（不编码中文）
        return `[${text}](?file=${relativePath})`;
    });
}
```

## 链接转换示例

### 输入（Markdown 原文）

```markdown
# 项目结构

- [整体项目结构](./01-项目结构.md)
- [多智能体协作](./03-多智能体协作/多智能体协作机制.md)
```

### 输出（处理后）

```markdown
# 项目结构

- [整体项目结构](?file=个人知识库/../../../clowder-ai架构wiki/01-项目结构.md)
- [多智能体协作](?file=个人知识库/../../../clowder-ai架构wiki/03-多智能体协作/多智能体协作机制.md)
```

## 处理流程

```
原始 Markdown
  → 正则匹配 [text](./path.md)
  → 解析相对路径
  → 转换为基于 Wiki 根目录的绝对路径
  → 生成 ?file=path 格式（不 URL 编码）
  → 返回处理后的 Markdown
```

## 关键设计决策

### 为什么不 URL 编码中文路径？

```javascript
// ❌ 错误：编码后前端无法正确解码
return `[${text}](?file=${encodeURIComponent(relativePath)})`;

// ✅ 正确：保持原始中文路径
return `[${text}](?file=${relativePath})`;
```

原因：
- 编码后的路径 `%E4%B8%AA%E4%BA%BA...` 在前端 `marked.js` 解析时会二次编码
- 导致实际请求的路径与服务器期望的不匹配
- 保持中文原样，前端直接传递即可
