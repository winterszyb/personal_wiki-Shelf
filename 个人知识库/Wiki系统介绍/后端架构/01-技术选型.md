# 后端架构 - 技术选型

## 技术栈

| 技术 | 用途 |
|------|------|
| Node.js | 运行时环境 |
| 原生 `http` 模块 | HTTP 服务器（零依赖） |
| 原生 `fs` 模块 | 文件读写 |
| 原生 `path` 模块 | 路径处理 |

## 为什么选择零依赖？

### 优势

1. **零配置**：无需安装 npm 依赖，直接运行
2. **轻量级**：整个后端只有一个 `server.js` 文件
3. **易于部署**：复制文件夹即可运行
4. **无版本冲突**：不用担心依赖包更新问题

### 文件结构

```
server.js
├── 路由分发器
│   ├── GET  /api/wiki/tree
│   ├── GET  /api/wiki/read
│   ├── POST /api/wiki/create
│   ├── PUT  /api/wiki/update
│   ├── DELETE /api/wiki/delete
│   └── PUT  /api/wiki/rename
├── 工具函数
│   ├── scanDirectory()      → 扫描目录树
│   ├── resolveMarkdownLinks() → 解析内部链接
│   └── getSafePath()        → 安全路径检查
└── 静态文件服务
    └── /index.html
```

## 核心模块

| 模块 | 函数 | 功能 |
|------|------|------|
| 目录扫描器 | `scanDirectory()` | 递归扫描目录结构 |
| 文件读取器 | `resolveMarkdownLinks()` | 读取并处理 Markdown 链接 |
| 安全检查 | `getSafePath()` | 防止目录遍历攻击 |

## 请求处理流程

```
HTTP 请求到达
  → 解析 URL 和方法
  → 路由匹配
  → 安全检查（getSafePath）
  → 文件系统操作
  → 返回 JSON 响应
```
