# 后端架构 - 目录扫描器

## 核心函数

```javascript
function scanDirectory(dir, baseDir = WIKI_DIR) {
    const items = [];
    const files = fs.readdirSync(dir);

    for (const file of files) {
        const fullPath = path.join(dir, file);
        const stat = fs.statSync(fullPath);
        // 统一使用正斜杠（Windows 兼容）
        const relativePath = path.relative(baseDir, fullPath).replace(/\\/g, '/');

        if (stat.isDirectory()) {
            const children = scanDirectory(fullPath, baseDir);
            if (children.length > 0) {
                items.push({
                    name: file,
                    type: 'directory',
                    path: relativePath,
                    children: children
                });
            }
        } else if (file.endsWith('.md')) {
            items.push({
                name: file.replace('.md', ''),
                type: 'file',
                path: relativePath,
                fullPath: fullPath.replace(/\\/g, '/')
            });
        }
    }

    // 排序：目录在前，文件在后，按名称排序
    items.sort((a, b) => {
        if (a.type === b.type) {
            return a.name.localeCompare(b.name);
        }
        return a.type === 'directory' ? -1 : 1;
    });

    return items;
}
```

## 扫描逻辑

```
Wiki 根目录
  → 遍历所有文件/目录
  → 跳过非 .md 文件
  → 递归扫描子目录
  → 构建树形结构
  → 排序（目录优先）
  → 返回 JSON
```

## 路径处理

### Windows 兼容

```javascript
// 将反斜杠转换为正斜杠
const relativePath = path.relative(baseDir, fullPath).replace(/\\/g, '/');
```

这是关键处理，确保：
- 前端 URL 使用正斜杠
- JSON 序列化不报错（反斜杠是转义字符）
- 跨平台兼容

## 性能考虑

- 每次请求 `/api/wiki/tree` 都重新扫描
- 对于小型知识库（<1000 文件）性能可接受
- 大型知识库可考虑缓存或增量更新
