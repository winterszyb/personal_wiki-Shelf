# 部署与扩展 - 数据存储

## 存储方案

采用**文件系统直接存储**，无需数据库：

```
D:\文档\wiki\                    ← Wiki 根目录
├── index.html                   ← 前端界面
├── server.js                    ← 后端服务器
├── package.json                 ← 项目配置
├── 启动服务器.bat               ← 一键启动
├── 停止服务器.bat               ← 一键停止
└── 个人知识库/                  ← 知识库内容
    ├── README.md
    └── ai-native-builder知识库/
          ├── Wiki系统介绍/
          │     ├── 总目录.md
          │     ├── 前端架构/
          │     ├── 后端架构/
          │     ├── 核心流程/
          │     └── 部署与扩展/
          └── clowder-ai架构wiki/
                └── ...
```

## 目录结构规范

```
知识库/
  ├── README.md        ← 入口文档
  ├── 章节A/
  │   ├── 01-xxx.md   ← 编号文档（保证排序）
  │   ├── 02-xxx.md
  │   └── 子章节/
  │       └── xxx.md
  └── 章节B/
      └── ...
```

## 文件命名规范

| 文件 | 命名规则 | 示例 |
|------|---------|------|
| 目录 | 中文或英文 | `前端架构/` `核心流程/` |
| 章节首页 | `README.md` 或 `总目录.md` | `总目录.md` |
| 编号文档 | `NN-名称.md` | `01-技术选型.md` |
| 普通文档 | `名称.md` | `API设计.md` |

## 优势

| 优势 | 说明 |
|------|------|
| **零配置** | 无需安装数据库 |
| **版本友好** | Git 可直接管理 |
| **便携性** | 整个知识库就是一个文件夹 |
| **通用性** | 任何文本编辑器都能打开 |
| **可读性** | 文件浏览器直接显示层级 |
