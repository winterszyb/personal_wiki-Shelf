# 部署与扩展 - 安全设计

## 路径安全检查

所有文件操作都经过 `getSafePath` 检查：

```javascript
function getSafePath(filePath) {
    const fullPath = path.join(WIKI_DIR, filePath);
    const resolvedPath = path.resolve(fullPath);
    const resolvedWikiDir = path.resolve(WIKI_DIR);

    if (!resolvedPath.startsWith(resolvedWikiDir)) {
        return null;  // 拒绝目录遍历攻击
    }

    return fullPath;
}
```

## 安全措施

### 1. 目录遍历防护

防止用户访问 Wiki 目录外的文件：

```
恶意请求: GET /api/wiki/read?file=../../../../etc/passwd
    ↓ getSafePath
权限拒绝: 403 Forbidden
```

### 2. 文件类型限制

只处理 `.md` 文件：

```javascript
// 目录扫描时过滤
if (file.endsWith('.md')) {
    items.push({ ... });
}

// 创建时自动添加扩展名
if (type === 'file' && !finalName.endsWith('.md')) {
    finalName += '.md';
}
```

### 3. 文件大小限制

Node.js 默认无限制，建议添加限制：

```javascript
const MAX_FILE_SIZE = 10 * 1024 * 1024;  // 10MB
const stat = fs.statSync(fullPath);
if (stat.size > MAX_FILE_SIZE) {
    res.writeHead(413);
    res.end(JSON.stringify({ error: 'File too large' }));
    return;
}
```

### 4. CORS 配置

```javascript
res.setHeader('Access-Control-Allow-Origin', '*');
res.setHeader('Access-Control-Allow-Methods', 'GET, POST, PUT, DELETE, OPTIONS');
res.setHeader('Access-Control-Allow-Headers', 'Content-Type');
```
