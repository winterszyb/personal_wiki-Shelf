# 会话续接

## 触发条件

当 Cat 的会话因为 Claude 上下文限制被封存时：
```
session_seal_requested 事件
  → CollaborationContinuityCapsule 提取
  → QueueProcessor.enqueueContinuation()
    → source: 'agent', autoExecute: true
    → Cat 被自动唤醒继续工作
```

## 会话封存场景

### 1. 上下文超限
- Claude 上下文窗口达到上限
- 触发 `session_seal_requested` 事件

### 2. 压缩边界
- 到达上下文压缩边界
- 系统自动封存当前会话

### 3. 手动封存
- Agent 主动请求会话封存
- 用于长期任务分段执行

## 续接流程

### CollaborationContinuityCapsule
```typescript
interface CollaborationContinuityCapsuleV1 {
  threadId: string;      // 哪个线程
  catId: string;         // 哪个 Cat
  mode: 'independent' | 'serial' | 'parallel';  // 续接模式
  ballState: 'in_progress' | 'completed' | 'needs_handoff' | 'needs_owner';
  seal: {
    sessionId: string;
    sessionSeq: number;
  };
}
```

### 自动续接
```
封存事件 → Capsule 提取 → Continuation 队列入队
  → autoExecute: true → 立即唤醒 → 继续工作
```

## 限制与保护

### 速率限制
- **每 (thread, cat) 对**: 60 分钟内最多 5 次续接
- **目的**: 防止无限循环续接

### 去重机制
- 使用 `continuationKey` 进行去重
- 相同续接请求不会重复入队

### 断路器
- 连续失败后触发断路器
- 暂停自动续接功能

## 续接提示

系统向 Agent 提供续接上下文：
```
Your previous session was sealed because of threshold_seal.
Continue the same structured work...
```
