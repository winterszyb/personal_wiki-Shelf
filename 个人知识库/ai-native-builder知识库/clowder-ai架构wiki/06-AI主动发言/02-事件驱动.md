# 事件驱动触发

## GitHub Webhook 实时触发

### 处理器位置
`GitHubRepoWebhookHandler.ts`

### 触发事件类型
- PR 开启/关闭
- Issue 创建/关闭
- PR Review 提交
- CI 检查状态变更

### 处理流程
```
GitHub 事件 → Webhook 接收 → HMAC 签名验证
  → 事件类型过滤 → invokeTrigger.trigger() 
  → Cat 被唤醒发言
```

## 轮询触发机制

### 1. CI/CD 状态轮询
**处理器**: `CiCdRouter`
**检查内容**: CI 构建状态变更
**触发条件**: CI 失败/成功状态变更

### 2. Merge 冲突检测
**处理器**: `ConflictRouter`
**检查内容**: Git 分支冲突状态
**触发条件**: 检测到新的冲突

### 3. Review 反馈轮询
**处理器**: `ReviewFeedbackRouter`
**检查内容**: PR Review 评论和修改
**触发条件**: 新的 Review 反馈

### 4. Issue 评论轮询
**处理器**: `IssueCommentTaskSpec`
**检查内容**: Issue 评论更新
**触发条件**: 新的 Issue 评论

## 事件处理特性

### 实时 vs 延迟
- **Webhook**: 实时触发，事件发生后立即响应
- **轮询**: 延迟触发，按预定间隔检查

### 认证与验证
- **HMAC 签名**: Webhook 请求验证 GitHub 来源
- **事件过滤**: 只处理白名单事件类型
- **权限检查**: 确保有权限访问相关仓库

## 典型应用场景

### 实时响应
- PR 创建时立即开始代码审查
- CI 失败时立即分析失败原因
- Review 提交时立即给予反馈

### 定期检查
- 每小时检查是否有新的 Merge 冲突
- 每天检查是否有新的 Issue 评论
- 每周检查 PR Review 状态
