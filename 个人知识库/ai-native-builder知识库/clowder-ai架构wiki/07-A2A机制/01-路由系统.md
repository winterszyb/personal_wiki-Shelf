# A2A 路由系统

## 整体状态机

```
用户消息 → 初始化 worklist = [A, B]
  → A 执行 → 输出扫描 → @C 被发现 → worklist.push(C)
  → B 执行 → 输出扫描 → 无 @ → 继续
  → C 执行 (A2A 来源) → 输出扫描 → 无 @ → 继续
  → worklist 耗尽 → 结束
```

## 三种路由路径

| 路径 | 触发方式 | 特点 |
|------|---------|------|
| **文本 @mention** | 行首 `@handle` | 串行接力，从输出文本解析 |
| **MCP post_message** | `cat_cafe_post_message` + `targetCats` | 回调路径，支持跨线程 |
| **MCP multi_mention** | `cat_cafe_multi_mention` | 并行分发 1-3 个目标，结果聚合返回 |

## 核心组件

### 1. 路由决策引擎
`routing-decision.ts` - 统一的路由守卫链
- 守卫链顺序: abort → depth → dedup → pingpong → fairness
- 输出: `RoutingDecision[]`

### 2. Worklist 管理
`WorklistRegistry.ts` - 可变工作列表管理
- 注册/注销工作列表
- 推送/查询工作列表项
- Ping-pong 追踪

### 3. 路由策略
- `route-serial.ts` - 串行路由（A2A 启用）
- `route-parallel.ts` - 并行路由（A2A 抑制）

## 路由决策类型

| 决策类型 | 含义 | 消费深度 |
|---------|------|---------|
| `enqueue_worklist` | 正常 A2A，推入工作列表 | 是 |
| `defer_queue` | 队列公平性延迟 | 是 |
| `mark_replyto` | 已在待处理，仅更新元数据 | 否 |
| `skip` | 跳过该 Cat | 否 |
| `block_pingpong` | Ping-pong 阻断 | 否 |
