# 智能体间通信

## 三种通信通道

### 方式一：A2A @Mention（文本级传递）
- 串行路由中，每个智能体输出完成后扫描文本中的行首 `@handle`
- 检测到的目标经过路由决策引擎校验后推入工作列表，形成接力链
- 限制：每次最多 @ 2 个目标，最大深度 15 层

### 方式二：MCP 回调工具
- 智能体可以调用 `cat_cafe_post_message`、`cat_cafe_multi_mention` 等 MCP 工具
- 通过 `{invocationId, callbackToken}` 进行 HTTP 回调认证（TTL 2 小时）

### 方式三：共享消息存储
- 所有智能体的输出都持久化为 `StoredMessage` 记录（Redis 存储）
- 增量上下文系统通过投递游标（delivery cursor）跟踪每个智能体已读消息边界

## 核心设计理念

这是一个**基于共享线程的异步消息传递架构**：智能体之间不直接通信，而是通过持久化的消息存储 + 文本解析（@mention）+ 结构化 MCP 工具三种通道协作。
