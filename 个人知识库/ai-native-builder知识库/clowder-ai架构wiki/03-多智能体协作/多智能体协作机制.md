# 多智能体协作机制

Clowder AI 的多智能体协作围绕 **"猫"（Cat）** 这个概念构建 —— 每个 Cat 是一个绑定到特定 LLM 提供商（Claude/GPT/Gemini 等）的命名人格化智能体。

## 整体流程

```
用户消息 → 路由决策 → 上下文组装 → 智能体执行 → A2A检测/回调 → 持久化
```

## 路由系统

位于 `packages/api/src/domains/cats/services/agents/routing/`，支持两种路由策略：

### 串行路由
- 智能体逐个响应，每个都能看到前者的输出
- 通过解析输出中的 `@handle` 实现智能体间传递（A2A），形成接力链

### 并行路由
- 所有智能体同时独立响应同一消息
- 通过 `mergeStreams()` 合并事件流

## 路由决策引擎

对每个候选智能体执行守卫链检查：
1. 中止 → 深度 → 去重 → ping-pong 检测 → 队列公平性 → 入队
