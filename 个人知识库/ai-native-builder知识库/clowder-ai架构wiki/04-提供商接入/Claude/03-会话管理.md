# Claude 会话管理

## 会话管理流程

`invoke-single-cat.ts` 中统一处理会话管理：

1. **会话获取**: 从 `sessionManager` 获取缓存的 sessionId
2. **交叉验证**: 与 `sessionChainStore` 交叉验证会话是否仍然活跃
3. **断路器保护**: 3 次连续恢复失败后触发断路器封存
4. **互斥锁**: 会话互斥锁防止并发恢复

## 会话模式

| 模式 | 实现方式 | 特点 |
|------|---------|------|
| **恢复模式** | `--resume <sessionId>` | 恢复已有会话，继续之前的工作 |
| **新会话模式** | 不提供 `--resume` | Claude CLI 自动分配新的 session ID |

## 会话 ID 提取

从 `system/init` 事件的 `session_id` 字段提取：
```javascript
// claude-ndjson-parser.ts:152-153
const sessionId = e.session_id;
```

## 会话跟踪

将会话 ID 捕获到 `metadata.sessionId`：
```javascript
metadata: {
  sessionId: extractedSessionId,
  // ... 其他元数据
}
```

## 断路器机制

- **最大重试次数**: 3 次
- **失败阈值**: 连续 3 次恢复失败
- **保护时间**: 失败后暂时封存该会话
