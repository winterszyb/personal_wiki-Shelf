# Codex 会话管理

## 会话管理流程

与 Claude 类似，`invoke-single-cat.ts` 中统一处理：

1. **会话获取**: 从 `sessionManager` 获取缓存的 sessionId
2. **交叉验证**: 与 `sessionChainStore` 交叉验证会话是否仍然活跃
3. **断路器保护**: 3 次连续恢复失败后触发断路器封存
4. **互斥锁**: 会话互斥锁防止并发恢复

## 会话模式

| 模式 | 实现方式 | 特点 |
|------|---------|------|
| **恢复模式** | `codex exec resume <sessionId>` | 恢复已有会话，继续之前的工作 |
| **新会话模式** | `codex exec` | 新建会话，包含沙箱模式和 .git 访问 |

## 会话 ID 提取

从 `thread.started` 事件的 `thread_id` 字段提取：
```javascript
// codex-event-transform.ts:37
const sessionId = e.thread_id;
```

## 会话跟踪

将会话 ID 捕获到 `metadata.sessionId`：
```javascript
metadata: {
  sessionId: extractedSessionId,
  // ... 其他元数据
}
```

## 新会话 vs 恢复会话

### 新会话特性
- 包含 `--sandbox <mode>` 沙箱模式
- 包含 `--add-dir .git` Git 目录访问
- 新建工作目录和上下文

### 恢复会话特性
- 不包含 `--sandbox` 和 `--add-dir`（仅新会话专属）
- 恢复之前的工作目录和上下文
- 延续之前的对话状态

## 断路器机制

- **最大重试次数**: 3 次
- **失败阈值**: 连续 3 次恢复失败
- **保护时间**: 失败后暂时封存该会话
