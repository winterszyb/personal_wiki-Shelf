## 项目简介

这是一个基于 Markdown 文件系统的个人知识库 Wiki 系统，采用前后端分离架构。无需数据库，所有内容以 Markdown 文件存储，通过 Web 界面进行浏览、编辑和管理。

### 为什么选择文件系统存储？

- **零配置**：无需安装数据库，clone 即用
- **Git 友好**：所有内容可版本管理
- **便携性强**：整个知识库就是一个文件夹
- **通用性好**：任何编辑器都能打开 Markdown 文件
- **迁移简单**：复制文件夹即可迁移

## 功能特性

### 核心功能
- **层级目录结构**：支持无限嵌套的目录树
- **Markdown 渲染**：完整的 Markdown 支持（表格、代码块、引用、列表等）
- **代码高亮**：支持多种编程语言语法高亮
- **CRUD 操作**：创建、读取、更新、删除文档
- **内部链接**：文档间通过相对路径互相跳转
- **图形化链接插入**：编辑器中可视化选择目标文件

### 界面功能
- **树状导航**：可展开/折叠的目录树
- **白天/黑夜主题**：CSS 变量方案，一键切换
- **响应式设计**：适配桌面和移动端
- **编辑器工具栏**：快捷插入 Markdown 格式
- **右键菜单**：便捷的重命名/删除操作

### 技术特性
- **零数据库**：纯文件系统存储
- **零依赖后端**：仅使用 Node.js 原生模块
- **前后端分离**：RESTful API 设计
- **安全防护**：路径遍历攻击防护

## 快速开始

### 环境要求

- Node.js >= 14.x

### 安装与启动

```bash
# 1. 克隆仓库
git clone https://github.com/winterszyb/personal_wiki-Shelf.git
cd knowledge-wiki

# 2. 启动服务器
node server.js

# 3. 打开浏览器
# 访问 http://localhost:3000
```

### 修改端口

编辑 `server.js` 第 7 行：

```javascript
const PORT = 3000;  // 改为你想要的端口
```

### 双击启动（Windows）

创建 `启动服务器.bat`：

```batch
@echo off
chcp 65001 > nul
cd /d "%~dp0"
node server.js
pause
```

## 系统架构

```
┌─────────────────────────────────────────────────────────┐
│                     浏览器（前端）                        │
│  ┌──────────┐  ┌──────────────┐  ┌─────────────────┐   │
│  │ 目录树导航 │  │ Markdown 预览 │  │   编辑器         │   │
│  └──────────┘  └──────────────┘  └─────────────────┘   │
└───────────────────────┬─────────────────────────────────┘
                        │ HTTP REST API
┌───────────────────────┼─────────────────────────────────┐
│                  Node.js 服务器（后端）                    │
│  ┌────────────────────────────────────────────────────┐ │
│  │ API 路由层                                          │ │
│  │ GET /api/wiki/tree    → 扫描目录树                   │ │
│  │ GET /api/wiki/read    → 读取文件                     │ │
│  │ POST /api/wiki/create → 创建文件/目录                 │ │
│  │ PUT /api/wiki/update  → 更新文件                     │ │
│  │ DELETE /api/wiki/delete → 删除文件/目录               │ │
│  │ PUT /api/wiki/rename  → 重命名                       │ │
│  └────────────────────────────────────────────────────┘ │
│                     ↕ 文件系统                           │
│            ┌──────────────────────┐                     │
│            │  Markdown 文件存储    │                     │
│            │  个人知识库/          │                     │
│            │    └── 子目录/       │                     │
│            │          └── *.md    │                     │
│            └──────────────────────┘                     │
└─────────────────────────────────────────────────────────┘
```

## API 文档

| 方法 | 路径 | 请求体 | 功能 |
|------|------|--------|------|
| GET | `/api/wiki/tree` | - | 获取完整目录树 |
| GET | `/api/wiki/read?file={path}` | - | 读取文件内容 |
| POST | `/api/wiki/create` | `{type, path, parentDir, content}` | 创建文件/目录 |
| PUT | `/api/wiki/update` | `{file, content}` | 更新文件内容 |
| DELETE | `/api/wiki/delete` | `{file}` | 删除文件/目录 |
| PUT | `/api/wiki/rename` | `{oldPath, newPath}` | 重命名 |

### 获取目录树

```
GET /api/wiki/tree
```

响应：
```json
{
  "tree": [
    {
      "name": "个人知识库",
      "type": "directory",
      "path": "个人知识库",
      "children": [...]
    }
  ]
}
```

### 创建文件

```
POST /api/wiki/create
Content-Type: application/json

{
  "type": "file",
  "path": "新文档.md",
  "parentDir": "个人知识库",
  "content": "# 标题\n\n内容..."
}
```

### 更新文件

```
PUT /api/wiki/update
Content-Type: application/json

{
  "file": "个人知识库/文档.md",
  "content": "# 新内容"
}
```

## 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| **前端** | 原生 HTML/CSS/JS | 无框架，直接浏览器运行 |
| | Marked.js (CDN) | Markdown 解析渲染 |
| | Highlight.js (CDN) | 代码语法高亮 |
| **后端** | Node.js | 运行时 |
| | http 模块 | HTTP 服务器（零依赖） |
| | fs 模块 | 文件系统操作 |
| | path 模块 | 路径处理 |
| **存储** | 文件系统 | Markdown 文件 |

## 开发历程

本项目经历了 8 个阶段的开发迭代：

1. **基础搭建**：服务器 + 前端界面 + Markdown 渲染 + 代码高亮

4. **内部链接**：链接转换 + 图形化插入 + 相对路径计算
5. **编辑增强**：工具栏 + 快捷插入 + 保存预览
6. **主题系统**：CSS 变量方案 + 白天/黑夜切换 + 持久化
7. **完善修复**：右键菜单 + 键盘快捷键 + 响应式设计
8. **文档体系**：多层级文档 + API 文档 + 问题记录
